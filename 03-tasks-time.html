Stage 03

Tasks & time

Before you edit, everything runs through Asana. Find your task, open the project Juan sends, and track your time against that task with TMetric.

The board

Every project is one board with three columns — the stage the work is in:

PrePrep — download the material.
PostThe edit — your work lives here.
DeliverExports that go to the client.

Each task is named prefix | name | version and carries a tag for the type of work:

  • D |Download · Pre — pull the material. No time tracking — just mark it done so the stage is clear.
  • S |Selections · Post — narrow the footage down.
  • E |Editing · Post — the first cut (v1).
  • AD |Adjustments · Post — revision rounds (v2, v3…).
  • D |Deliver · Deliver — the client export.

Each task

  1. Open your task in AsanaRead the brief in the description and note the due date.
  2. Open the project fileJuan attaches the .drp with everything loaded. Save it in your project's 01_Project/01_Resolve and open it — the proxies relink.
  3. Start the timer from the taskHit the green Start timer on the Asana task. The first time you work a project, pick the project in TMetric so the entry files correctly.
  4. Keep TMetric + Asana open in the browserCheck task · tag · project are right, and stop/play the timer as you work.
  5. Track only Selections & EditingDownload isn't timed. Time runs on the Selections, Editing and Adjustments tasks — the real work.

One-time setup

Do this once, then you're set for every project.

1 · TMetric Desktop (Mac)
Watches idle time in the background. In settings, set Detect idle time → Ask me with a 5 minute timeout. If you step away that long, it asks what to do with the time — keep, discard, or mark as a break.

2 · TMetric browser extension
Install it, then turn on the Asana integration (TMetric → Integrations → Asana). Refresh Asana — a green Start timer button appears on every task.

Remember

+Read the brief before you touch anything.
+Keep TMetric + Asana open in the browser the whole session.
–Don't trust the timer blindly — wrong task / tag / project means time logged in the wrong place.
